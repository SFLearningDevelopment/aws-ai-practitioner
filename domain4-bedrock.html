Amazon Bedrock

Domain 4 · AIF-C01 · 11 Key Concepts
Domain 4
Concept Definition
Amazon Bedrock Fully managed service to access foundation models via API — no infrastructure management required. Supports models from Anthropic, Meta, Cohere, Mistral, Stability AI, and Amazon Titan.
Amazon Titan AWS's own family of foundation models. Includes Titan Text (LLM), Titan Embeddings, and Titan Image Generator.
Bedrock Knowledge Bases Managed RAG solution. Connect data sources (e.g. S3), auto-embed & store in a vector DB, then query via API.
Easiest way to build RAG on AWS — fully managed
Bedrock Agents Orchestrate multi-step tasks by letting an FM autonomously call APIs and tools to complete complex goals.
Bedrock Guardrails Define content filters, PII redaction, and topic denial lists to make FM applications safer and compliant.
Model Customization (Bedrock) Two options:
Fine-tuning — train on labeled input/output examples
Continued Pre-training — train on raw unlabeled domain data
Provisioned Throughput Purchase dedicated model capacity in Bedrock for consistent, predictable performance at scale (vs. on-demand pricing).
Model Evaluation (Bedrock) Built-in tool to evaluate and compare FM outputs using automatic metrics or human review workflows.
Amazon Bedrock Studio Browser-based UI for experimenting with FMs, building prompts, and prototyping Bedrock applications — no code needed.
Amazon Q AWS's generative AI-powered assistant. Two variants: Amazon Q Business (enterprise Q&A over company data) and Amazon Q Developer (AI coding assistant for developers).
Exam tip: Amazon Q Developer can explain, debug, and generate AWS code
PartyRock Amazon Bedrock's free no-code playground for building and sharing generative AI apps in a browser — no AWS account needed.
Think: quick prototyping and experimentation with FMs

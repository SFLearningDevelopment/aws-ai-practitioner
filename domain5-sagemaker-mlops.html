Amazon SageMaker & MLOps

Domain 5 · AIF-C01 · 13 Key Concepts
Domain 5
Concept Definition
Amazon SageMaker End-to-end ML platform: data prep → build → train → tune → deploy → monitor. One of the most exam-heavy topics.
SageMaker Studio Web-based IDE for the full ML lifecycle. Notebooks, experiments, pipelines, and deployment — all in one place.
SageMaker Autopilot AutoML feature — automatically tries many algorithms and hyperparameters to find the best model for your data.
SageMaker Canvas No-code ML UI for business analysts. Build and deploy models with no programming required.
SageMaker Ground Truth Managed data labeling service. Uses human workers (via Mechanical Turk or private workforce) + automated labeling.
SageMaker Pipelines CI/CD for ML workflows. Automate the steps from data prep to model deployment as a repeatable pipeline.
SageMaker Model Monitor Detects model quality issues in production — data drift, concept drift, bias, and prediction quality degradation.
SageMaker Feature Store Centralized repository to store, share, and reuse ML features across teams and models.
SageMaker Clarify Detects bias in data and models, and provides explainability reports (SHAP values). Exam favourite.
MLOps DevOps principles applied to ML: automate training, versioning, testing, deployment, and monitoring of models.
Model Registry Catalog of trained model versions with metadata, approval status, and deployment history.
Data Drift Input data distribution shifts over time, causing model performance to degrade. Requires retraining.
Detected by: SageMaker Model Monitor
Concept Drift The relationship between inputs and outputs changes over time (world changes, model becomes stale).

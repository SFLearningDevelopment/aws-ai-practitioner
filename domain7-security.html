Security, Compliance & Governance

Domain 7 · AIF-C01 · 11 Key Concepts
Domain 7
Concept Definition
AWS Shared Responsibility Model AWS secures the cloud infrastructure (hardware, networking, facilities).
You secure what you put in the cloud (data, access controls, configuration).
IAM (Identity & Access Management) Controls who can access AWS services and what actions they can perform. Uses users, groups, roles, and policies.
AWS KMS Key Management Service — create and manage encryption keys. Used to encrypt data at rest across AI/ML services.
VPC (Virtual Private Cloud) Isolated network environment in AWS. Used to keep ML workloads private and secure from the public internet.
Data Encryption At rest: data stored encrypted (S3, RDS, EFS)
In transit: data moving over the network encrypted (TLS / HTTPS)
AWS CloudTrail Logs all API calls across your AWS account. Used for audit trails, compliance reporting, and incident investigation.
Amazon Macie Uses ML to automatically discover, classify, and protect sensitive data (PII) in S3 buckets.
PII (Personally Identifiable Info) Data that can identify an individual (name, SSN, email, IP). Must be protected.
AWS tools: Amazon Comprehend, Amazon Macie, Bedrock Guardrails
Compliance Frameworks AWS supports GDPR, HIPAA, SOC 2, ISO 27001, and more. AI workloads handling regulated data must meet these standards.
AWS Artifact Self-service portal for on-demand access to AWS compliance reports, certifications, and agreements.
Model Governance Policies, processes, and tools to manage the AI model lifecycle — approval workflows, version control, and audit logs.

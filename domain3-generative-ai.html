Generative AI Fundamentals

Domain 3 · AIF-C01 · 18 Key Terms
Domain 3
Term Definition
Generative AI AI that creates new content (text, images, audio, code) rather than just classifying or predicting.
Foundation Model (FM) Large pre-trained model trained on massive datasets; adaptable to many tasks via fine-tuning or prompting.
Examples: Claude, Llama, Titan, Stable Diffusion
Large Language Model (LLM) Foundation model specializing in text. Trained on billions of tokens to predict the next token.
Token The basic unit an LLM processes — roughly a word or sub-word. Pricing and context limits are per-token.
Prompt Input text sent to a generative model. Quality of output is highly dependent on prompt quality.
Prompt Engineering Designing prompts to guide model outputs. Key techniques: zero-shot, few-shot, chain-of-thought.
Zero-shot Prompting Model answers with no examples provided in the prompt — relies purely on pre-trained knowledge.
Few-shot Prompting Provide a few input→output examples in the prompt so the model learns the pattern in-context.
Chain-of-Thought (CoT) Prompting technique that asks the model to reason step-by-step before answering. Improves complex tasks.
Context Window Maximum number of tokens a model can process at once (input + output). Larger = more context retained.
Temperature Controls randomness of output.
Low (0) = deterministic/focused · High (1+) = creative/varied
Hallucination When a model confidently generates false or fabricated information. A key risk of LLMs.
RAG (Retrieval-Augmented Generation) Combines an LLM with a retrieval step — grounds answers in real documents, reducing hallucinations.
Flow: Query → Retrieve docs → Augment prompt → Generate answer
Fine-tuning Further training a pre-trained model on domain-specific data to specialize its behavior.
More expensive than prompting; better for consistent style or task
Embedding Numerical vector representation of text (or data) capturing semantic meaning. Used in search & RAG.
Vector Database Stores embeddings for fast similarity search. Core component of RAG pipelines.
AWS options: Amazon OpenSearch, pgvector on Aurora
Diffusion Model Generative model that creates images by learning to reverse a noise-addition process.
Examples: Stable Diffusion, Amazon Titan Image Generator
Multimodal Model Model that processes multiple input types (text, image, audio). Can also generate across modalities.
